反馈回路

系统最基本的控制单元 — 输出返回影响输入

负反馈（平衡回路）
偏差 → 检测 → 修正 → 趋向目标。恒温器原理：温度偏离设定点 → 传感器检测 → 加热/制冷 → 回归设定点。回路中有奇数个负号。系统趋向稳定——但如果有延迟，可能出现振荡。

可行系统模型 VSM

Stafford Beer — 任何可存活系统必须包含的5个子系统

S5
政策 / 身份
平衡内外、定义系统存在的意义
  • 平衡 S3（内部优化）和 S4（外部情报）的矛盾
  • 在根本性危机中重新定义系统身份
  • 回答"我们是谁？我们为什么存在？"
  • 类比：CEO / 董事会 / 宪法
S4
情报 / 未来
监测外部环境，建模未来
  • 系统性扫描外部环境的变化和威胁
  • 建模未来可能的场景
  • 将发现传递给 S3 和 S5 做决策
  • 类比：战略研究部 / R&D / 市场调研
S3
控制 / 优化
全局资源分配，监控运营绩效
  • 优化 S1 之间的资源分配
  • 监控各 S1 的绩效指标
  • 对 S1 下达指令并接收报告
  • 类比：COO / 运营管理层
S3*
审计通道
绕过常规报告的直接检查
  • 突击检查 S1 的实际状态
  • 发现 S1→S3 常规报告中的信息失真
  • 防止"报喜不报忧"
  • 类比：内审 / 随机抽查 / 暗访
S2
协调
消除 S1 之间的振荡和冲突
  • 标准化接口，减少 S1 之间的摩擦
  • 调度和排期，防止资源争抢
  • 协调是自动的、基于规则的（不是人治）
  • 类比：调度系统 / 标准化协议 / SOP
S1
运营单元
执行价值创造的基本单元（递归：每个S1也是完整VSM）
  • 直接产出价值的业务单元
  • 需要足够的自主权处理日常干扰
  • 递归原则：每个 S1 内部也包含完整的 S1-S5
  • 类比：业务部门 / 产品线 / 团队
S3 ↔ S1 指令 / 报告通道
S4 ↔ 外部 环境扫描通道
S3 ↔ S4 内外平衡通道
S1 ↔ S1 横向协调通道

Ashby 必要多样性定律

"Only variety can absorb variety." — 控制器多样性必须 ≥ 干扰多样性

干扰多样性 V(D)
60
vs
控制器多样性 V(R)
40
V(R) < V(D) — 控制必然失败！缺口：20
怎么办？
路径A 放大器：增加控制器多样性 — 赋予更多权限/工具/自动化
路径B 衰减器：降低干扰多样性 — 标准化、过滤、缓冲
路径C 组合：同时使用（通常最优）

四大 Agent

控制论分析的四个专项智能体

◉
系统建模师
System Modeler
绘制反馈回路、因果回路图、存量流量模型，识别系统动力学结构
因果回路图 存量-流量 延迟分析 杠杆点
  1. 系统素描 — 列出所有变量（存量/流量/辅助）
  2. 因果连接 — 标注极性(+/-)和延迟
  3. 回路分析 — 识别每个R/B回路的增益和状态
  4. 主导结构 — 哪个回路当前主导？主导权会转移吗？
  5. 杠杆点 — 区分症状和结构性原因
⚙
调节器
Regulator
设计控制机制，运用Ashby定律和VSM进行治理架构设计
Ashby定律 VSM PID思维 衰减/放大器
  1. 控制需求诊断 — 被控对象、干扰源、现有控制器
  2. Ashby多样性审计 — V(R) ≥ V(D)?
  3. 多样性管理 — 放大器 / 衰减器 / 组合
  4. VSM架构检查 — S1-S5完整性
  5. 控制机制设计 — 传感器-比较器-执行器
▢
信号分析师
Signal Analyst
分析信息流、噪声、熵、沟通断裂，运用信息论和二阶控制论
信息流映射 Shannon信道 黑箱方法 二阶控制论
  1. 信息流映射 — 谁→什么→谁，画出拓扑
  2. 信道评估 — 容量/延迟/噪声/冗余
  3. 病理诊断 — 断裂/延迟/衰减/噪声/过载/失真
  4. 黑箱分析 — 不透明子系统的输入-输出建模
  5. 二阶检查 — 观察者效应、盲点、权力-信息纠缠
◆
活力架构师
Viability Architect
评估系统存活力和韧性，运用VSM递归分析和适应层级理论
VSM递归 韧性审计 适应层级 自组织
  1. 系统身份 — 为什么存在？边界？环境？
  2. VSM完整性 — S1-S5 + S3* + 4条信道
  3. 适应层级 — 一阶(参数)/二阶(结构)/三阶(范式)
  4. 韧性审计 — 冗余/多样性/模块化/连通性
  5. 自组织条件 — 必要条件和障碍检查
